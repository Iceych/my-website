三、typescript VS javascript

TypeScript 是 Microsoft 开发和维护的一种面向对象的编程语言。它是 JavaScript 的超集，包含了 JavaScript 的所有元素，可以载入 JavaScript 代码运行，并扩展了 JavaScript 的语法。JavaScript + 更多功能 = TypeScript。

语言 开发者 类型 特性 编译 数据绑定 社区 热度趋势
Typescript Microsoft in 2012 强类型 支持静态类型，强大而直观 需要编译 利用接口和类型等概念来定义所使用的数据 有一个较小的软件开发者社区


Javascript Brendan Eich (Netscape)，1995 年 弱类型 简单，优化了代码的兼容性，易于阅读和编写 不需要编译 无 拥有庞大的软件开发人员社区

结论：业务开发用javascript，组件开发用typescript。经实践，typescript开发需要大量的类型声明，否则就是大量的any的使用（已经失去了使用typescript的意义），比javascript开发耗时约多30%。业务开发时性价比不大，组件开发时可提升组件可读性和易用性。

四、less VS scss

LESS和SCSS都是css的预处理器，为css开发提供了变量，运算，继承，嵌套等功能，用一种专门的编程语言为 CSS 增加了一些编程的特性，使代码更加的便于阅读和维护。

语言 创建者 语法 性能 社区
Scss Hampton Catlin于2006年 缩进式语法 缓存机制避免重复编译 更大更活跃的社区，更多的插件与工具
Less Alexis Sellier于2009年 CSS语法 每次修改后重新编译 活跃的社区

结论：less。less与scss的语法、支持的功能差别不大，但是less的学习曲线相对更短，考虑到公司开发人员的技术储备，故选择less。

六、组件共享&项目管理方式选型

主要是针对业务高度相似的平台组件共享的问题的解决方案，如下：（参考）

方案 基本定义 方案图解 优点 缺点
CV大法 直接从一个项目复制到另一个项目

速度快 可维护性低，各个项目各自多了一套代码，当需求发生变更时需要各自更新

Npm包

被共享的代码作为npm包，由引用方通过npm install安装使用

  • 常见，简单易上手；
  • 版本控制功能，使组件的功能、样式对每个项目都是可控的；
  • 每个应用都会打包该模块，导致依赖的包冗余，没有真正意义上的共享复用
  • 当npm包进行更新发布了，应用还需要重新构建，调试麻烦且低效
Monorepo(单体式仓库) 允许我们将多个项目放到同一个仓库里面进行管理。主张不拆分repository，而是在单仓库里统一管理各个模块的构建流程、版本号等

  • 多项目代码都在一个仓库中，相同版本依赖提升到顶层只安装一次，节省磁盘内存，可以避免大量的node_module冗余
  • 代码复用高，方便进行代码重构
  • 工程配置一致，代码质量标准及风格也很容易一致
  • 仓库体积过大，维护成本高

module federation（模块联邦）

 直接将一个应用的包以CDN的方式共享给其他项目应用于另一个应用，同时具备整体应用一起打包的公共依赖抽取能力

  • 模块实时更新
  • 资源共享&版本对比，相同资源不需要重复加载
  • 去中心化，应用/模块既可以是引用者也可以是提供者
  • 没有样式隔离机制，可能造成样式污染
  • 项目之间有强依赖关系，其中一个项目崩溃直接影响所有依赖它的其他项目

结论：npm包：用于与业务无关的公共组件；Monorepo（单体式仓库）：用于包含多个子项目的业务开发（如，活动的主分会场）；module federation（模块联邦）：用于业务型组件共享；
